# System Design Concepts — Quiz Log

> One file per service. Created by `/concept-quiz` after each sub-feature implementation.
> Reference: "Designing Data-Intensive Applications" — Martin Kleppmann

Each file tracks:
- **Concept Coverage table** — concepts tested, best score, last tested date
- **Session log** — every quiz run with questions, user answers, correct answers, and explanations

Concepts scored ≥ 80% are deprioritised in future quiz sessions for that service.
