# Incident Log

> Real bugs and unexpected behaviours encountered while building the platform.
> Each entry maps the failure to a system design concept from DDIA or related literature.
> Created by `/incident`. These are learning records, not production post-mortems.

Severity guide:
- **High** — data loss, privacy violation, or SLO breach caught in test harness
- **Medium** — test failure, wrong output, functional regression
- **Low** — unexpected behaviour caught before any test ran
