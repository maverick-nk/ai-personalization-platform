# Retrospective — privacy

**Date:** 2026-04-30
**Service:** privacy

---

## What went well

Planning protocol and stack choices upfront — HTTP vs gRPC for the internal endpoint, opt-in vs opt-out consent default, ORM selection — prevented backtracking during implementation. Making those decisions before writing a line of code meant every subsequent choice had a clear rationale to lean on.

## Would design differently

Settle the retention policy and its compliance implications at the schema design phase, not after the table is already written. The 6-month → 1-month → 3-month iteration happened because the compliance tradeoff discussion (CCPA 12-month window, GDPR Article 7(1), hot/cold tiering) came after the default was already in the config. Starting with "what does the audit log need to prove, and for how long?" would have landed at 3 months + ADR 0006 in one step.

## Most surprising

`expire_on_commit=False` on `AsyncSessionLocal`. Async SQLAlchemy silently breaks attribute access after `commit()` without this flag — accessing `record.updated_at` post-commit triggers a lazy load that raises `MissingGreenlet` with no obvious connection to the flag being missing. The error message points nowhere near the root cause, making it a trap that's easy to hit and hard to diagnose without prior experience.

## Open questions

How the Go gRPC inference-api will call the HTTP privacy check end-to-end. The latency contract is clear (<5ms), and the endpoint is defined, but the mechanics of an HTTP client inside a gRPC service, how errors are handled when the privacy service is unavailable, and whether a circuit breaker is warranted — all of this is still abstract until Step 5.

## Follow-up actions

- **Open question about inference-api wiring** → will be answered directly when Step 5 (inference-api) is implemented. Flag it as the first design decision to settle when starting that service: what happens when the privacy service is unreachable — fail open (allow personalization) or fail closed (return fallback)? That's a compliance decision worth an ADR.
- **expire_on_commit=False surprise** → maps to SQLAlchemy session lifecycle and async context management. Run `/concept-quiz` on the privacy service to test understanding of connection pooling, session scoping, and lazy loading in async ORMs.
- **Retention policy iteration** → consider opening any future compliance-adjacent service (e.g. model-training audit trail) with an explicit "what does this log need to prove?" question before writing the schema.
