🧪 Test Flow: Watch Event → Updated Recs

1Test posts POST /events/watch (user_id, content_id, watch_pct)
2Event ingested into Kafka with pseudonymized ID
3Flink computes updated features within ~2s
4Redis key updated for user
5Test calls GET /recommend/{user_id}
6Assert new category appears in top-N results

🔒 Test Flow: Consent Revocation

1Test calls PATCH /privacy/consent/{user_id} → revoke
2Postgres consent record updated immediately
3Test calls GET /recommend/{user_id}
4Privacy middleware intercepts, blocks Redis feature fetch
5Assert response is fallback trending feed (non-personalized)
6Assert audit log recorded the revocation event

❄️ Test Flow: Cold Start User

1Test registers new user_id with no history
2GET /recommend/{user_id} — Redis cache miss
3Inference API detects missing features → triggers fallback
4Assert generic trending content returned (no personalization)
5Post 3 watch events → assert recs begin to shift toward watched genres

⚡ Test Flow: Feature Freshness

1Record timestamp T0 before posting watch event
2Post event · Poll GET /features/{user_id}
3Assert feature key updated within <5 seconds of T0
4Check Prometheus metric: feature_age_seconds < threshold

🔁 Test Flow: Model Hot-Swap

1Trigger training pipeline on latest Parquet snapshot
2New model registered in Model Registry with version tag
3Assert Inference API polls and loads new model within poll interval
4Assert no downtime — prior requests still served during swap
5Assert feature schema of new model matches Redis keys

📊 Test Flow: Latency Validation

1Run k6 load test: 100 concurrent users requesting recommendations
2Assert p95 latency < 50ms end-to-end
3Assert Redis fetch < 5ms at p99
4Assert no consent check adds >5ms overhead
5Grafana dashboard confirms no latency spikes under load
